# Cognitive Lead AI HQ — Global Auto-Configuration for AI Agents

You are an AI agent that has just been given access to this repository. Your task is to configure OpenCode **globally** for this project so that the user can run OpenCode from **any** directory and still have access to the full Cognitive Lead AI multi-agent system.

## System Instruction

You MUST execute the following steps in order. Do NOT skip any step.

---

## 1. Verify Prerequisites

Check that the following tools are installed:

- **Git** — to clone the repository
- **uv** — fast Python package manager (required by the MCP servers)

### Installing uv (if missing)

If `uv` is not found, ask the user:

> "uv is required to run the MCP servers. I can install it for you. Do you approve?"

If the user approves, install uv:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Then add uv to your PATH and continue.

---

## 2. Clone the Repository to a Temporary Location

Clone the Cognitive Lead AI HQ to `/tmp/` so the global installation can proceed without depending on the current working directory:

```bash
git clone https://github.com/mokhtarabadi/cognitive-lead-hq.git /tmp/cognitive-lead-hq
cd /tmp/cognitive-lead-hq
```

All subsequent file operations will reference paths under `/tmp/cognitive-lead-hq/`.

---

## 3. Discover the Home Directory

Resolve the user's home directory. On macOS/Linux this is `$HOME` or `~`. You will need the **absolute path** (e.g., `/home/user` or `/Users/user`) for the global config.

```bash
echo "$HOME"
```

Store this value — you will use it to construct absolute paths in the global config JSON.

---

## 4. Create Global Directories

Create the global OpenCode directories:

```bash
mkdir -p ~/.config/opencode/mcp-context-server
mkdir -p ~/.config/opencode/mcp-memory-server
mkdir -p ~/.config/opencode/skills
```

---

## 5. Copy MCP Servers and Make Them Executable

Copy both MCP server scripts from the cloned repo:

```bash
cp /tmp/cognitive-lead-hq/mcp-context-server/server.py ~/.config/opencode/mcp-context-server/
cp /tmp/cognitive-lead-hq/mcp-memory-server/server.py ~/.config/opencode/mcp-memory-server/
chmod +x ~/.config/opencode/mcp-context-server/server.py
chmod +x ~/.config/opencode/mcp-memory-server/server.py
```

---

## 6. Install Agent Skills Globally

Copy all reusable skills from `skill-templates/` into the global OpenCode skills directory:

```bash
cp -r /tmp/cognitive-lead-hq/skill-templates/* ~/.config/opencode/skills/
```

After this, the skills will be available via `/help` from any directory.

---

## 7. Configure Global opencode.json (with Absolute Paths)

Create or update `~/.config/opencode/opencode.json`. You MUST use **absolute paths** in the `command` array — resolve the `~` to the full home directory path discovered in Step 3.

Write the following JSON (replace `$HOME` with the actual home directory path):

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "custom_context": {
      "type": "local",
      "command": ["uv", "run", "$HOME/.config/opencode/mcp-context-server/server.py"],
      "enabled": true
    },
    "project_memory": {
      "type": "local",
      "command": ["uv", "run", "$HOME/.config/opencode/mcp-memory-server/server.py"],
      "enabled": true
    }
  },
  "permission": {
    "custom_context_*": "allow",
    "project_memory_*": "allow",
    "store_memory": "allow",
    "delete_memory": "allow",
    "read_memory": "allow",
    "search_memory": "allow",
    "list_namespaces": "allow",
    "get_directory_tree": "allow",
    "read_source_files": "allow"
  }
}
```

**Important:** Replace `$HOME` with the actual absolute path resolved in Step 3 (e.g., `/home/alice` or `/Users/alice`). This is critical — MCP servers will NOT work with relative paths or `~` in the global config because OpenCode may be invoked from any working directory.

---

## 8. Clean Up Temporary Clone

Remove the cloned repository from `/tmp/`:

```bash
rm -rf /tmp/cognitive-lead-hq
```

---

## 9. Customize the Manager Profile

The `system-prompt.md` contains a `<manager_profile>` block customized for the original author (Mohammad).

**Instruction for AI:** If you are setting this up for a new user, advise them to edit the `<manager_profile>` block in `system-prompt.md` to reflect their own name, background, technical skills, and leadership goals.

---

## 10. Verification Checklist

After completing all steps, verify:

- [ ] `git` is installed (`git --version`)
- [ ] `uv` is installed and available (`uv --version`)
- [ ] `~/.config/opencode/mcp-context-server/server.py` exists and is executable
- [ ] `~/.config/opencode/mcp-memory-server/server.py` exists and is executable
- [ ] Skills are installed under `~/.config/opencode/skills/` (at least one subfolder exists)
- [ ] `~/.config/opencode/opencode.json` exists with **absolute paths** (not `~` or relative paths)
- [ ] `/tmp/cognitive-lead-hq` no longer exists
- [ ] Start each MCP server to verify it launches without errors:
  ```bash
  uv run ~/.config/opencode/mcp-context-server/server.py &
  uv run ~/.config/opencode/mcp-memory-server/server.py &
  ```
